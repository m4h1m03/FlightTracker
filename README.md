# Flight Tracker

A personal data project to practice the full data analyst pipeline end-to-end:
ingest live data from a public API, store it in a relational database, transform
it with SQL, and visualize it on a dashboard.

## Stack

- **Language:** Python (extending PY4E foundation)
- **Database:** PostgreSQL (local install via Postgres.app)
- **DB GUI:** DBeaver
- **Data source:** OpenSky Network API (free, real-time flight positions)
- **Dashboard:** Looker Studio (free, browser-based, native Mac, free Postgres connector)
- **Optional second dashboard:** IBM Cognos (while lab access lasts during cert)

## Goals

- Build a real ETL pipeline (Extract -> Transform -> Load) on data I collected myself
- Practice intermediate SQL on data that didn't come pre-cleaned
- Ship a live dashboard that I can show on a CV or in interviews
- Bridge what I learned in PY4E with what the IBM cert teaches

## Skills demonstrated

- Python: API calls, JSON parsing, error handling, scheduled execution
- SQL: schema design, joins, aggregates, window functions
- Database: normalization, indexing basics, connection management
- BI tools: dashboard design, KPI selection, visual storytelling
- Engineering hygiene: README, version control, environment management
